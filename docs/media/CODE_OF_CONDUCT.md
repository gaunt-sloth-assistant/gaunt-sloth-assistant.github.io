# Code of Conduct

## Our Standard

We expect contributors and maintainers to keep project interactions professional, respectful, and constructive.

Examples of expected behavior:

- Be respectful in discussions, reviews, and issue threads
- Assume good intent, while still being clear and technically rigorous
- Give actionable feedback focused on the code, design, or process
- Accept that people may have different levels of experience
- Help keep discussion relevant to the project

Examples of unacceptable behavior:

- Harassment, personal attacks, insults, or discriminatory language
- Trolling, deliberate provocation, or sustained disruption
- Publishing private information without explicit permission
- Abusive or hostile conduct in Issues, pull requests, reviews, or other project spaces
- Spam or clearly bad-faith contributions

## Scope

This Code of Conduct applies to project spaces, including:

- GitHub Issues
- Pull requests and review comments
- Discussions connected to this repository

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct. Maintainers may also temporarily or permanently restrict participation when necessary to protect the project and its contributors.

## Reporting

If you experience or witness unacceptable behavior, report it to the maintainers through GitHub. If the matter should not be reported publicly, contact the repository owner directly instead of opening a public Issue.

When reporting, include:

- A description of the incident
- Links to relevant comments, issues, or pull requests
- Any additional context that would help review the report

Reports should be made in good faith. Maintainers will review them and respond as appropriate.
